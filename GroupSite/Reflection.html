Tailwind CSS Reflection

Using Tailwind made it easier for our group to style each page without writing a lot of custom CSS. We were able to quickly build layouts for our navbar, cards, and sections just by using utility classes. Overall, it helped us keep the design consistent and match the OU look we wanted for ReviewOU.

Major Tailwind Features Used

  1. Flexbox for layout – We used flex classes on the navbar and other sections to keep items lined up, spaced out, and centered on the page. 
    <nav class="flex items-center justify-between">
  ...
</nav>
  2. Grid for responsive cards – The team member cards and other card sections use grid classes so the layout adjusts automatically on small and large screens. 
    <div class="grid gap-6 sm:grid-cols-2 lg:grid-cols-3">
  ...
</div>
  3. Responsive breakpoints – Classes like sm: and lg: helped us change font sizes and layouts so the site looks good on phones, tablets, and laptops. 
    <h1 class="text-3xl sm:text-4xl">...</h1>
  4. Utility classes for color, spacing, and shadows – We used utilities like bg-white, p-6, rounded-xl, and shadow-md to style cards and sections without needing extra CSS files. 
    <article class="bg-white p-6 rounded-xl shadow-md border border-gray-200">
  ...
</article>

Tailwind vs. Bootstrap

Group Tailwind Rating (1–10): 8 / 10

Our group liked Tailwind because it gave us a lot of control over the look of each page and made it easy to keep a consistent style across the site. It was also nice that we could handle most of the design just by adding classes in the HTML instead of writing long CSS files, even though remembering all the class names took some practice.

Compared to Bootstrap, Tailwind felt more flexible and less “pre-styled". While Bootstrap is better if you want fast, ready-made components like buttons and navbars. Bootstrap can be quicker for simple layouts, but Tailwind worked better for making ReviewOU feel more customized and specific to OU.
